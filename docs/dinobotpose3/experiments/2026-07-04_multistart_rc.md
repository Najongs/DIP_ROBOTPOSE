# 2026-07-04 — 멀티스타트 RC + SAM-IoU basin 선택 ❌ 반증

## 가설
남은 실패(orb 잔여 −0.010, 40% 가림 붕괴)의 공통 근원 = **wrong basin**: init 포즈가 틀린 회전 분지에 있으면 보수적 refine이 탈출 불가. 6월 MCL 반증의 원인은 *학습 selector*였고, 지금은 **SAM-IoU라는 외부 증거 선택기**가 있으므로 멀티가설이 성립.

## 설계 (`rc_refine_from_dump.py --multi-start`)
- 가설 군: init 카메라 포즈의 **base-Z축 회전 섭동** δ∈{0,±30°,±60°} — 2D 키포인트가 구분 못 하는 게이지 방향(단축/base-yaw 모호성), 실루엣은 구분 가능
- 각 가설 독립 refine → 최종 SAM-IoU 최고 채택, 원 가설 우선 마진 0.01 (do-no-harm)

## 판정 — 가설 기각, 그러나 진단 가치 큼
1. **orb 잔여는 wrong-basin이 아님** — rot head가 이미 basin을 고정(6월 +0.117의 목적). 전환할 분지가 없음. → orb 공격 방향은 detector 2D(로드맵 ④ crop 해상도)로 확정.
2. **40% 붕괴도 R-basin이 아님** — 전환은 10% 프레임에서 발화했지만 ADD 불변: 손상이 상류(θ/2D 자체 붕괴, pose 0.315)라 어느 회전에서 시작해도 RC가 복구 불가. → head 레벨 처방(T1/T2 가림-증강 학습)이 정답 경로임을 역으로 입증.
3. 코드는 `--multi-start` 플래그(기본 off)로 유지 — 무해 확인됐고, 향후 rot head 없는 구성에서 유용 가능.
