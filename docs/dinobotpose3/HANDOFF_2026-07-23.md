## 1. 공정성 감사 결론 (사용자 지적으로 발견, `_debate_tmp/kuka_gate/audit_logs/`)

레퍼런스(RoboPEPP/HoRoPose/DREAM)는 **전체 테스트셋 + 모든 키포인트 + deterministic**로 평가. 우리와 비교:
- ✅ ADD-AUC 공식 동일. ✅ KUKA/Baxter 완전 결정론적(seed 무관, bit-identical). ✅ Panda held-out-30%는 full보다 ~1pt 더 어려움(보수적, 유리하지 않음).
- ❌ **문제 1 — metric mask**: 논문 KUKA/Baxter 수치(`u1_solver_vs_direct.py`)가 `valid_mask`(화면안 키포인트만) 평균 = RoboPEPP의 all-keypoints보다 관대. 교정 시 **KUKA 69.0→67.6, Baxter 71.3→68.7**(둘 다 결정론적).
- ❌ **문제 2 — Baxter 키포인트 셋(치명)**: 우리=**왼팔 7kp**(left_s0..w2), RoboPEPP=**전신 17kp**. 비교 불가 → **17-kp 전신 재학습으로 공정화 중**(§3).

## 3. Baxter 17-kp 전신 재학습 (진행 중 — 이어서 완료)

**목표**: RoboPEPP와 동일한 17 키포인트로 Baxter 재학습 → all-keypoints ADD로 공정 비교.
17 kp = `torso_t0` + 왼팔 8(s0,s1,e0,e1,w0,w1,w2,hand) + 오른팔 8(동일) (`RoboPEPP/ijepa/src/datasets/dream.py:154`).

- **FK 완성됨**: `TRAIN/model_v4.py:662` `baxter_forward_kinematics` (17 kp 출력). ⚠️ **먼저 FK가 GT 17 키포인트를 sub-mm로 재현하는지 검증**(과거 데이터-면책과 동일 방식) — 안 되면 그 키포인트 원인부터.
- **검출기 학습 중**: `TRAIN/outputs_heatmap/baxter_fullbody_detector_20260723_132444` (17채널). Ep1 2D AUC 0.29, L2 꼬리 ~125px = **양팔+말단 혼동으로 전신이 매우 어려움**(예상대로).
- **이어서 할 것**: 검출기 수렴 → 그 위에 angle head(양팔 14관절) + rot head 학습 → `baxter_add_eval.py`를 17-kp로 확장해 all-keypoints ADD 평가(full 5982, solver+true-K, deterministic).
- **정직한 전망**: 전신 17-kp는 RoboPEPP 34.4 수준으로 떨어질 가능성 큼. 그럼 **Baxter "1위" 주장 철회** → "동일 알고리즘이 전신 Baxter로 확장, SOTA와 대등/근접"으로 서술. (핵심 서사 = Panda real SOTA + 하나의 알고리즘 이식성.)

학습 명령 템플릿(왼팔 버전에서 --keypoint-names 17개로 확장):
```bash
# 검출기: TRAIN/train_heatmap.py --data-dir .../baxter_synth_train_dr --keypoint-names torso_t0,left_s0,...,right_hand
# 각도:   TRAIN/train_angle.py   (frozen 17-kp detector 위, angle-joints=양팔 14)
# 회전:   TRAIN/train_rotation.py
# GPU: UUID 지정(CLAUDE.md). free memory 기준.
```

